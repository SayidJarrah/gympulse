# Bug Brief: import — oversized CSV shows generic "Import failed." instead of specific size-limit message

Date: 2026-03-27 23:36
Reported by: e2e-tester

## Failing Test
Spec file: `frontend/e2e/class-schedule.spec.ts`
Test name: `AC 43 — importing a CSV file larger than 2 MB shows IMPORT_FILE_TOO_LARGE error`

## Symptom
Expected: `page.getByText('The file exceeds the 2 MB size limit.')` to be visible after uploading a
2,097,252-byte CSV and clicking Upload.

Observed: The ImportModal transitions to an error state showing only the generic text "Import failed."
with no specific message about the file size. The `role=menu` drop zone hint "CSV file — max 2 MB"
is present before upload but the app does not perform any client-side size validation on file
selection, and the backend returns a non-specific error rather than the IMPORT_FILE_TOO_LARGE code
the spec expects.

## Browser Evidence
Screenshot: `screenshots/20260327-233624/02-import-file-too-large.png`

The screenshot shows the ImportModal in its error state: a red circle-X icon followed by the text
"Import failed." and a "Try Again" button. There is no mention of the 2 MB limit or the error code
IMPORT_FILE_TOO_LARGE. The server returned an HTTP error (logged in console as
"Failed to load resource: the server responded with a status of..." for
`http://localhost:3000/api/v1/admin/schedule/import`).

## Suspected Layer
The backend endpoint `/api/v1/admin/schedule/import` does not appear to return a structured
IMPORT_FILE_TOO_LARGE error response body. The frontend ImportModal component receives the failure
but only renders a generic "Import failed." message rather than reading an error code from the
response and surfacing the appropriate human-readable text.

This could be a backend issue (no size limit enforced / no structured error returned for oversized
payloads), a frontend issue (ImportModal does not map an IMPORT_FILE_TOO_LARGE error code to a
user-facing message), or both. The missing client-side pre-upload size check is also a frontend gap.

## Suggested Agent
@frontend-dev (add client-side file size check before upload that rejects files >2 MB with the
message "The file exceeds the 2 MB size limit." and never sends the request to the backend)
@backend-dev (ensure the endpoint returns `{ "error": "...", "code": "IMPORT_FILE_TOO_LARGE" }`
when the multipart payload exceeds 2 MB so the frontend can surface the message correctly)

## Reproduction Steps
1. Log in as `admin@gymflow.local` / `Admin@1234`.
2. Navigate to `/admin/scheduler`.
3. Click "Import".
4. Select any file larger than 2 MB (e.g. a 2,097,252-byte CSV of repeated 'a' characters).
5. Click "Upload".
6. Observe: modal shows "Import failed." — the specific size-limit message is never shown.
