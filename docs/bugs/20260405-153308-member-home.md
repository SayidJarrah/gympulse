# Bug Brief: member-home — three critical copy and layout failures

Date: 2026-04-05 15:33

## Overview

Manual inspection of `/home` via Playwright MCP (with a valid authenticated session) found three critical failures: two copy divergences and one layout order inversion. All three are from the gap report's "Design Divergence" and "Broken Flows" sections. The spec `frontend/e2e/member-home.spec.ts` contains tests that assert the correct behaviour for each.

---

## Bug 1: Hero renders below membership section, not above it (AC-04)

### Failing Test
Spec: `frontend/e2e/member-home.spec.ts`
Test name: `AC-04 hero renders before the membership section in DOM order`

### Failure (expected from Playwright once global-setup is fixed)
```
expect(heroBox.y).toBeLessThan(memberBox.y)
// heroBox.y = ~530  (hero is second — below viewport)
// memberBox.y = ~70  (membership card is first — at top of page)
```

### Steps to Reproduce
1. Log in as any `USER` account.
2. Navigate to `/home`.
3. Observe the page top.

### What Happens
The membership section ("Your access" / plan name card) renders as the first element in `<main>`. The hero ("Welcome back") renders after it.

### What Should Happen
The design spec (`docs/design/member-home.md`) and PRD AC 4 both require the hero (`MemberHomeHero`) to be the first primary content section, with the membership card below it.

### Evidence
Screenshots: `screenshots/20260405-133000/01-home-no-membership.png`
Screenshots: `screenshots/20260405-133000/02-home-active-membership.png`

The screenshots confirm the membership card ("Activate your access" / "Trial Week") is the topmost content element. The "Welcome back" hero is clearly below it.

Console errors: none
Network: all API calls returned 200 or 404-as-empty (expected for no-membership)

### Severity
Critical — page layout contradicts both the PRD and design spec. The hero is a primary orientation landmark.

## Bug 2: No-active-membership headline reads "Activate your access" instead of "No active membership" (AC-07)

Spec: `frontend/e2e/member-home.spec.ts`
Test name: `AC-07 no-active-membership state shows "No active membership" headline`

### Failure (expected)
```
expect(page.locator('#membership').getByRole('heading', { name: 'No active membership' })).toBeVisible()
// Actual heading text in DOM: "Activate your access"
```

### Steps to Reproduce
1. Log in as a `USER` with no active membership.
